# タブバー z-order 修正 + 設定画面の別ウィンドウ分離 — 最終仕様（Result）

> 生成日: 2026-03-02
> 検証モード: フル検証（5/5 タスク完了）

## 機能概要

エディタ（VSCode, Cursor, Zed）のサブウィンドウ（設定画面、モーダルパネル等）が開いてもタブバーが隠れないよう、タブバーウィンドウの z-order を NSWindowLevel=9（NSModalPanelWindowLevel より上）に設定する。あわせて、タブマネージャの設定画面を別ウィンドウとして分離し、設定表示中もタブバーが維持されるようにした。実装中に発覚した5件の不具合対応により、他アプリ切替時のタブバー表示判定にウィンドウサイズベースの条件を追加している。

## 仕様からの変更点

### 他アプリに切り替えた際のタブバー表示判定が、一律非表示からウィンドウサイズベースの条件付き判定に変更された

- **plan.md**: エディタ以外のアプリに切り替えた際は、従来通りタブバーを画面外に退避する（一律非表示）
- **実装**: 切り替え先アプリのウィンドウ幅が画面幅の 85% 以上の場合のみタブバーを退避。85% 未満の小さいウィンドウ（macOS 設定など）ではタブバーを表示したまま維持する
- **経緯**: troubleshoot 1〜4 で発覚した循環問題（「一律非表示」と「一律表示」の間で振れ続ける）を解決するため、ウィンドウサイズベースの判定を導入した（[troubleshoot-2026-03-02-4.md](./troubleshoot-2026-03-02-4.md) 参照）
- **技術詳細**: `observer.rs` で AX API (`ax_helper.rs`) を使ってフォーカスアプリの最大ウィンドウサイズを取得し、`is_large_window` フラグとしてフロントエンドに通知。フロントエンド (`useAppLifecycle.ts`) で `is_large_window === true` の場合のみ `setPosition(0, -10000)` で退避

### 初回起動アプリに対するウィンドウ検出のリトライ機構が追加された

- **plan.md**: 記載なし
- **実装**: Dock からアプリを初回起動した場合、ウィンドウがまだ作成されていない可能性があるため、バックエンド側で最大 4 回 × 500ms のリトライ（cold start recheck）を実行し、ウィンドウが出現した時点でサイズ判定を再実施する。フロントエンド側でもエディタ起動時に最大 8 回 × 500ms のウィンドウ一覧取得リトライを実行する
- **経緯**: troubleshoot-5 で発覚。初回起動時は macOS の通知がウィンドウ作成より先に来るため、リトライが必要（[troubleshoot-2026-03-02-5.md](./troubleshoot-2026-03-02-5.md) 参照）
- **技術詳細**: `observer.rs` の `schedule_cold_start_recheck` で DEBOUNCE_VERSION チェック付きリトライ。`useAppLifecycle.ts` の `coldStartRetryVersion` で重複リトライを防止

### ルーティング方式が URL パラメータからウィンドウラベルに変更された

- **plan.md**: URL パラメータ `?view=settings` でタブバーと設定画面を切り替える
- **実装**: Tauri のウィンドウラベル（`getCurrentWindow().label === "settings"`）で切り替える
- **経緯**: Tauri のマルチウィンドウでは各ウィンドウにラベルが自動付与されるため、URL パラメータより自然な分岐方式。ユーザーから見た振る舞いに違いはない
- **技術詳細**: `App.tsx` でウィンドウラベルを判定。バックエンド (`lib.rs`) は設定ウィンドウ作成時に `index.html` をそのまま読み込み（パラメータなし）

## ロジック

### 仕様

- タブバーウィンドウの z-order は NSWindowLevel=9（NSModalPanelWindowLevel=8 より上、Dock=20・メニュー=24 より下）
- z-order の設定はアプリ起動時に 1 回のみ実施（動的変更なし）
- 設定画面は別ウィンドウ（ラベル: `settings`、600x600 固定、リサイズ不可、画面中央、通常 z-order）
- 設定ウィンドウが既に存在する場合は前面化、未作成なら新規作成
- 設定画面の閉じるボタンはウィンドウ自体を破棄する
- エディタがアクティブになるとタブバーを表示位置 (0, 0) に復帰
- 他アプリがアクティブになった際の判定: ウィンドウ幅 >= 画面幅の 85% → タブバー退避、85% 未満 → 表示維持
- ウィンドウサイズ取得不能時（コールドスタート）はタブバー表示を維持し、最大 4 回 × 500ms でリトライ
- エディタのコールドスタート時はウィンドウ一覧取得を最大 8 回 × 500ms でリトライ
