# 不具合調査: ターミナルなどのウィンドウを開くとタブバーが隠れる

調査日: 2026-03-02

## 関連する過去の調査

- 前回: [troubleshoot-2026-03-02-1.md](./troubleshoot-2026-03-02-1.md)
- 前回の修正方針: 動的ウィンドウレベル切り替え（editor時: level 9, other時: level 0）
- 前回の修正が効かなかった理由: `set_window_level(0)` に降格すると、macOS の z-order によりアクティブなアプリのウィンドウが前面に来るため、タブバーが背後に隠れる。「自然な z-order に任せれば見える」という前提が誤りだった。

## 症状

パターン: リグレッション（前回の修正が効いていない）

- **操作手順**: エディター使用中にターミナルなどのウィンドウを開く
- **期待される動作**: タブバーが表示されたまま
- **実際の動作**: タブバーが隠れる

## 原因

`set_window_level(0)` でタブバーを通常ウィンドウ (NSWindowLevel=0) に降格すると、macOS の z-order によりアクティブなアプリ（ターミナル等）のウィンドウが前面に来る。ターミナルのウィンドウは画面上部のタブバー領域 (y=0, 高さ36px) も覆うため、タブバーが見えなくなる。

実行フロー:
  1. ターミナルがアクティベート → `app_type: "other"` で emit (`observer.rs`)  ← ✓
  2. `useAppLifecycle.ts:417-418` → `set_window_level(0)` 実行  ← ★ 問題箇所
  3. タブバーが NSWindowLevel=0 に降格 → ターミナルウィンドウの後ろに隠れる  ← ✗

根拠: NSWindowLevel=0 のウィンドウは、アクティブなアプリのウィンドウの背後に回る。ウィンドウサイズに関係なく z-order 上は隠れる。前回の「小さいウィンドウなら隠れない」という前提は macOS のウィンドウ管理の仕組みと合っていなかった。

## 修正方針

### コード修正

- `useAppLifecycle.ts:417-418` の `invoke("set_window_level", { level: 0 })` を削除
- `useAppLifecycle.ts:378-379` の `invoke("set_window_level", { level: 9 })` を削除（setup 時の一度だけで十分）
- `restore_window_positions` の呼び出しは残す（エディターウィンドウのオフセット復元は必要）
- タブバーは常に NSWindowLevel=9 で表示を維持する

- 「タブバーは常に画面上端に表示される」に変更

なし
