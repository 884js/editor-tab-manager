# 不具合調査: Slack 使用中にタブバーが表示されてしまう

調査日: 2026-03-02

## 関連する過去の調査

- 前回: [troubleshoot-2026-03-02-2.md](./troubleshoot-2026-03-02-2.md)
- 前回の修正方針: タブバーを常に NSWindowLevel=9 で表示（画面外移動を廃止）
- 前回の修正が引き起こした問題: 非エディターアプリ（Slack 等）使用時にもタブバーが最前面に表示され続ける

## 症状

パターン: 期待 vs 実際（前回の修正の副作用）

- **操作手順**: Slack を使用中
- **期待される動作**: タブバーが隠れている
- **実際の動作**: タブバーが Slack の上に表示されている

## 原因

troubleshoot-2 の修正で `app_type === "other"` 時の画面外移動 (`setPosition(0, -10000)`) を削除し、タブバーを常に NSWindowLevel=9 で表示する方針にした。その結果、非エディターアプリ使用時にもタブバーが最前面に表示され続ける。

実行フロー:
  1. Slack がアクティベート → `app_type: "other"` で emit  ← ✓
  2. `useAppLifecycle.ts:349-360` の else ブロック  ← ✓
  3. `restore_window_positions` のみ実行、タブバーを隠す処理なし  ← ★ 問題箇所
  4. タブバーが NSWindowLevel=9 のまま画面上端に残る  ← ✗

根拠: 3回の調査で設計の矛盾が明確になった:
- 画面外移動あり → エディターの設定画面でも隠れる（troubleshoot-1 で報告）
- level 下げ → ターミナルでも隠れる（troubleshoot-2 で報告）
- 何もしない → Slack でも出る（今回）

しかし troubleshoot-1 の「macOS システム設定で隠れる」は実際には正常動作。
macOS システム設定はエディターのサブウィンドウではなく別アプリなので、
タブバーが隠れるのは正しい。エディターのサブウィンドウ（設定画面、検索等）は
エディターのプロセス内で動作するため app_type === "editor" のままであり、
NSWindowLevel=9 により上に表示される。

つまり元の画面外移動の動作 + NSWindowLevel=9 の組み合わせが正解。

## 修正方針

### コード修正

- `useAppLifecycle.ts` の `app_type === "other"` ハンドラに `setPosition(0, -10000)` を復活させる
- `isVisibleRef.current = false` も復活させる

### plan.md の修正

不要。元の AC2「他のアプリに切り替えるとタブバーが隠れる」は正しかった。

### デバッグログの除去

なし
